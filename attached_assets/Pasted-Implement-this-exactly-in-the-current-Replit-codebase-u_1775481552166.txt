Add authenticated topical-content sync endpoints so Vercel can publish to /blog/:slug.

Required endpoints (same handler):
- POST /api/sync/blog

- Compare to process.env.SYNC_SECRET

    "content_type": "...",
    "priority_score": 0,
    "search_volume": 0,
    "keyword_difficulty": 0
  },
  "destination": "blog",
  "url": "/blog/<slug>"

Rules:
1) Validate payload/content/slug; return 400 on missing values.
2) Upsert by slug:
   - existing slug => update post
   - new slug => create post
3) Map fields into blog post store:
   - title/headline/slug/meta_description/summary/full_article
   - faq/sources/internal_links
   - topic metadata
4) Publication behavior:
   - if content.status === "published": publish/visible
   - else keep as draft/unpublished
5) Return 200 JSON:
{
  "success": true,
  "slug": "<slug>",
  "url": "https://cryptokiller.org/blog/<slug>",
  "post_id": "<id>",
  "action": "created|updated"
